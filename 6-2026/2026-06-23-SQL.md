# XX. Or

## 1. What is OR?

`OR` is used to combine multiple conditions in a `WHERE` clause.

Simply put:

Only one condition needs to be true.

## 2. How is it Different from AND?

### AND

```sql
WHERE condition_1
AND condition_2
```

=> All conditions must be true.

### OR

```sql
WHERE condition_1
OR condition_2
```

=> Only one condition needs to be true.

## 3. Example

```sql
SELECT *
FROM movies
WHERE year > 2014
OR genre = 'action';
```

Meaning:

Retrieve movies that:

* Were released after 2014.
* OR belong to the action genre.

## 4. Understanding OR

Suppose there is a movie:

| year | genre  |
| ---- | ------ |
| 2020 | comedy |

Condition 1:

```text
year > 2014
```

=> True

Condition 2:

```text
genre = 'action'
```

=> False

=> The movie is still selected because one condition is true.

Another example:

| year | genre  |
| ---- | ------ |
| 2010 | action |

Condition 1: False

Condition 2: True

=> Still selected.

Another example:

| year | genre  |
| ---- | ------ |
| 2020 | action |

Condition 1: True

Condition 2: True

=> Selected.

Another example:

| year | genre  |
| ---- | ------ |
| 2010 | comedy |

Condition 1: False

Condition 2: False

=> Not selected.

## 5. Exercise from the Lesson

```sql

OR genre = 'action';
```

Result:

* All action movies.
* All movies released after 2014.
* Movies that are both action and released after 2014 are also selected.


Meaning:

* romance

to be displayed.

## 7. Comparing AND and OR

### AND

```sql
WHERE genre = 'romance'
AND year > 2014
```

=> Must be a romance movie and released after 2014.

### OR

```sql
WHERE genre = 'romance'
OR year > 2014
```

=> Only needs to be a romance movie or released after 2014.

### Things to Remember

* `OR` is used to combine multiple conditions.
* Only one condition needs to be true.
* Syntax:

```sql
WHERE condition_1
OR condition_2
```

* `AND` = All conditions must be true.
* `OR` = One condition being true is enough.

Example:

```sql
SELECT *
FROM movies
WHERE genre = 'romance'
OR genre = 'comedy';
```

# XXI. Order By

## 1. What is ORDER BY?

`ORDER BY` is used to sort data.

Simply put:

Display data in a specific order.

Examples:

* Sort names from A → Z.
* Sort ratings from highest → lowest.
* Sort years from oldest → newest.

## 2. Why Do We Need ORDER BY?

Suppose the data is:

| name    | year |
| ------- | ---- |
| Titanic | 1997 |
| Avatar  | 2009 |
| Frozen  | 2013 |

The data is unsorted.

We can use `ORDER BY` to arrange it.

## 3. Syntax

```sql
SELECT *
FROM table_name
ORDER BY column_name;
```

Meaning:

* `ORDER BY` = Sort.
* `column_name` = Column used for sorting.

## 4. Example

```sql
SELECT *
FROM movies
ORDER BY name;
```

Meaning:

Sort by the `name` column.

Result:

| name    |
| ------- |
| Avatar  |
| Frozen  |
| Titanic |

From A → Z.

## 5. What is ASC?

`ASC` (Ascending) means ascending order.

Example:

```sql
SELECT *
FROM movies
ORDER BY name ASC;
```

Result:

* A → Z
* 0 → 9
* Low → High

In practice:

```sql
ORDER BY name
```

and

```sql
ORDER BY name ASC
```

produce the same result.

`ASC` is the default.

## 6. What is DESC?

`DESC` (Descending) means descending order.

Example:

```sql
SELECT *
FROM movies
ORDER BY year DESC;
```

Result:

* Newest → Oldest
* Highest → Lowest
* Z → A

Example:

| year |
| ---- |
| 2023 |
| 2020 |
| 2015 |
| 2010 |

## 7. Example from the Lesson

```sql
SELECT *
FROM movies
WHERE imdb_rating > 8
ORDER BY year DESC;
```

Meaning:

### Step 1

```sql
WHERE imdb_rating > 8
```

Filter movies with IMDb ratings greater than 8.

### Step 2

```sql
ORDER BY year DESC
```

Sort those movies from newest to oldest.

## 8. ORDER BY Always Comes After WHERE

Correct:

```sql
SELECT *
FROM movies
WHERE imdb_rating > 8
ORDER BY year;
```

Incorrect:

```sql
SELECT *
FROM movies
ORDER BY year
WHERE imdb_rating > 8;
```

## 9. Exercise 1


Meaning:

* Retrieve the `name` and `year` columns.
* Sort movie names from A → Z.

## 10. Exercise 2

```sql

FROM movies
ORDER BY imdb_rating DESC;
```

Meaning:

* Retrieve `name`, `year`, and `imdb_rating`.
* Sort by IMDb rating from highest to lowest.

### Things to Remember

* `ORDER BY` is used to sort data.
* `ASC` = Ascending.
* `DESC` = Descending.
* `ORDER BY` comes after `WHERE`.
* You can sort by:

  * Name
  * Year
  * IMDb Rating
  * Any other column

Examples:

```sql
ORDER BY name
```

→ A to Z

```sql
ORDER BY imdb_rating DESC
```

→ Highest to lowest

# XXII. Limit

## 1. What is LIMIT?

`LIMIT` is used to restrict the number of rows displayed.

Simply put:

Retrieve only a certain number of results.

## 2. Why Do We Need LIMIT?

Suppose a table contains:

```text
100,000 rows of data
```

If we use:

```sql
SELECT *
FROM movies;
```

SQL displays all rows.

This can be difficult to view.

Instead, we can use:

```sql
SELECT *
FROM movies
LIMIT 10;
```

Only the first 10 rows are displayed.

## 3. Syntax

```sql
SELECT *
FROM table_name
LIMIT number;
```

Meaning:

* `LIMIT` = Restrict.
* `number` = Maximum number of rows.

## 4. Example

```sql
SELECT *
FROM movies
LIMIT 5;
```

Meaning:

Display only the first 5 rows.

## 5. LIMIT Always Comes Last

Correct:

```sql
SELECT *
FROM movies
LIMIT 10;
```

Correct:

```sql
SELECT *
FROM movies
WHERE imdb_rating > 8
ORDER BY imdb_rating DESC
LIMIT 10;
```

Incorrect:

```sql
SELECT *
FROM movies
LIMIT 10
ORDER BY imdb_rating DESC;
```

## 6. Combining ORDER BY and LIMIT

Example:

```sql
SELECT *
FROM movies
ORDER BY imdb_rating DESC;
```

Meaning:

* Sort movies by IMDb rating from highest to lowest.

If we add LIMIT:

```sql
SELECT *
FROM movies
ORDER BY imdb_rating DESC
LIMIT 3;
```

Meaning:

* Sort from highest rating to lowest.
* Retrieve only the first 3 movies.

=> The 3 highest-rated movies.

### Things to Remember

* `LIMIT` is used to restrict the number of displayed rows.
* Syntax:

```sql
LIMIT number
```

* It is often used with `ORDER BY`.

# XXIII. Case

## 1. What is CASE?

`CASE` is used to create different results based on conditions.

Simply put:

`CASE` in SQL is similar to `if-else` in programming.

## 2. Why Do We Need CASE?

Suppose we have IMDb ratings:

| name    | imdb_rating |
| ------- | ----------- |
| Movie A | 9.0         |
| Movie B | 7.0         |
| Movie C | 4.0         |

We want to display:

* Fantastic
* Poorly Received
* Avoid at All Costs

instead of displaying only numbers.

This is where `CASE` is useful.

## 3. Syntax

```sql
CASE
   WHEN condition THEN result
   WHEN condition THEN result
   ELSE result
END
```

Meaning:

* `WHEN` = If.
* `THEN` = Then.
* `ELSE` = Otherwise.
* `END` = End of the CASE statement.

## 4. Example

```sql
SELECT name,
CASE
   WHEN imdb_rating > 8 THEN 'Fantastic'
   WHEN imdb_rating > 6 THEN 'Poorly Received'
   ELSE 'Avoid at All Costs'
END
FROM movies;
```

How it works:

If:

```text
imdb_rating > 8
```

=> Display:

```text
Fantastic
```

If:

```text
imdb_rating > 6
```

=> Display:

```text
Poorly Received
```

If no condition is satisfied:

```text
Avoid at All Costs
```

## 5. Example Result

| name    | Review             |
| ------- | ------------------ |
| Movie A | Fantastic          |
| Movie B | Poorly Received    |
| Movie C | Avoid at All Costs |

## 6. Using AS to Rename the Column

```sql
SELECT name,
CASE
   WHEN imdb_rating > 8 THEN 'Fantastic'
   WHEN imdb_rating > 6 THEN 'Poorly Received'
   ELSE 'Avoid at All Costs'
END AS 'Review'
FROM movies;
```

Meaning:

```text
Review
```

will be the new column name.

## 7. Exercise from the Lesson

Requirement:

* Chill if the genre is romance.
* Chill if the genre is comedy.
* Intense in all other cases.

Statement:

```sql
SELECT name,
CASE
   WHEN genre = 'romance' THEN 'Chill'
   WHEN genre = 'comedy' THEN 'Chill'
   ELSE 'Intense'
END AS 'Mood'
FROM movies;
```

## 8. Result

Suppose:

| name    | genre   |
| ------- | ------- |
| Titanic | romance |
| Frozen  | comedy  |
| Avatar  | action  |

Result:

| name    | Mood    |
| ------- | ------- |
| Titanic | Chill   |
| Frozen  | Chill   |
| Avatar  | Intense |

## 9. How to Read CASE

```sql
CASE
   WHEN genre = 'romance' THEN 'Chill'
   WHEN genre = 'comedy' THEN 'Chill'
   ELSE 'Intense'
END
```

Read as:

* If romance → Chill.
* If comedy → Chill.
* If neither of the above → Intense.

### Things to Remember

* `CASE` is similar to `if-else`.
* `WHEN` = Condition.
* `THEN` = Result when the condition is true.
* `ELSE` = Default result.
* `END` = End of the CASE statement.
* You can use `AS` to rename the column.

Example:

```sql
CASE
   WHEN genre = 'romance' THEN 'Chill'
   WHEN genre = 'comedy' THEN 'Chill'
   ELSE 'Intense'
END AS 'Mood'
```

# XXV. Count

## 1. What is COUNT()?

`COUNT()` is used to count the amount of data.

Simply put:

`COUNT()` = Count how many rows of data there are.

## 2. Syntax

```sql
SELECT COUNT(*)
FROM table_name;
```

Meaning:

* `COUNT()` = Count.
* `*` = All rows.
* `FROM` = From which table.

## 3. Example

```sql
SELECT COUNT(*)
FROM fake_apps;
```

Meaning:

* Count all apps in the `fake_apps` table.

Suppose the table is:

| id | name  |
| -- | ----- |
| 1  | App A |
| 2  | App B |
| 3  | App C |

Result:

| COUNT(*) |
| -------- |
| 3        |

There are 3 rows of data.

## 4. What Does COUNT(*) Mean?

```sql
COUNT(*)
```

Meaning:

Count all rows in the table.

This is the most common way to use COUNT().

## 5. Combining with WHERE

We can filter and count at the same time.

Example:

```sql
SELECT COUNT(*)
FROM fake_apps
WHERE price = 0;
```

Meaning:

* Select only free apps.
* Then count those apps.


How many apps are in the `fake_apps` table?

```sql

FROM fake_apps
WHERE price = 0;
```

Read as:

How many free apps are there?

## Things to Remember

* `COUNT()` is used to count data.
* `COUNT(*)` counts all rows.
* It can be combined with `WHERE` to count based on conditions.

Examples:

```sql
SELECT COUNT(*)
FROM fake_apps;
```

```sql
SELECT COUNT(*)
FROM fake_apps
WHERE price = 0;
```

## Lesson Summary

* `COUNT()` is used to count the amount of data.
* `COUNT(*)` counts all rows in a table.
* `WHERE` can be used to count only rows that satisfy a condition.
* This is the first Aggregate Function and one of the most commonly used functions in SQL.

`SUM()` is used to calculate the total of values in a column.

`SUM()` = Add all values together.


* `FROM` = From which table.

```sql

FROM fake_apps;
```

* Retrieve all values from the `downloads` column.

```sql
FROM fake_apps;
```

```text
600
```


* Syntax:

```sql

FROM table_name;
```

Example:

```sql
SELECT SUM(downloads)
FROM fake_apps;
```

## Lesson Summary

* `SUM()` adds all values in a column together.
* It is commonly used to calculate total downloads, revenue, or other numeric data.
* Example:

```sql
SELECT SUM(downloads)
FROM fake_apps;
```

returns the total downloads of all apps in the table.

# XXIX. Round

## 1. What is ROUND()?

`ROUND()` is used to round numbers.

Simply put:

`ROUND()` makes results easier to read.

## 2. Why Do We Need ROUND()?

Example:

```text
2.02365
```

This result is quite long.

We can round it to:

```text
2.02
```

or

```text
2
```

## 3. Syntax

```sql
ROUND(column_name, number_of_decimal_places)
```

Where:

* `column_name` = Column to round.
* The second number = Number of digits after the decimal point.

## 4. Example

```sql
SELECT ROUND(price, 0)
FROM fake_apps;
```

Meaning:

* `price` = App price.
* `0` = Do not keep decimal places.

Example:

| price |
| ----- |
| 2.3   |
| 4.7   |
| 8.1   |

Result:

| ROUND(price, 0) |
| --------------- |
| 2               |
| 5               |
| 8               |

## 5. Round to 2 Decimal Places

```sql
SELECT ROUND(price, 2)
FROM fake_apps;
```

Example:

| price   |
| ------- |
| 2.02365 |

Result:

```text
2.02
```

## 6. Exercise 1

Display the app name and the rounded price.

```sql

FROM fake_apps;
```

Meaning:

* Display the `name` column.
* Display the rounded price.

## 7. Exercise 2

Before:

```sql
SELECT AVG(price)
FROM fake_apps;
```

Result:

```text
2.02365
```

Round it to 2 digits:

```sql
SELECT ROUND(AVG(price), 2)
FROM fake_apps;
```

Result:

```text
2.02
```

## 8. ROUND() and AVG() Can Be Used Together

Example:

```sql
SELECT ROUND(AVG(price), 2)
FROM fake_apps;
```

SQL will:

Step 1: `AVG(price)`

Calculate the average.

Step 2: `ROUND(..., 2)`

Round the result.

## Things to Remember

* `ROUND()` is used to round numbers.
* Syntax:

```sql
ROUND(column_name, number_of_decimal_places)
```

* `0` = Do not keep decimal digits.

```sql
ROUND(price, 0)
```

* `2` = Keep 2 digits after the decimal point.

```sql
ROUND(price, 2)
```

* It can be combined with `AVG()`.

```sql
SELECT ROUND(AVG(price), 2)
FROM fake_apps;
```

## Lesson Summary

* `ROUND()` is used to round numbers.
* The second parameter decides the number of digits after the decimal point.
* `ROUND(price, 0)` rounds to a whole number.
* `ROUND(price, 2)` keeps 2 decimal places.
* It is often used with `AVG()` to make results easier to read.

# XXX. Group By I

## 1. What is GROUP BY?

`GROUP BY` is used to group similar data into groups.

Simply put:

Divide data into groups and then calculate for each group.

## 2. Why Do We Need GROUP BY?

Suppose there is data:

| category | downloads |
| -------- | --------- |
| Games    | 1000      |
| Games    | 2000      |
| Music    | 500       |
| Music    | 1500      |

We want to know:

The total downloads for each category.

This is where `GROUP BY` is used.

## 3. Without GROUP BY

To calculate the average for each year:

```sql
SELECT AVG(imdb_rating)
FROM movies
WHERE year = 1999;
```

```sql
SELECT AVG(imdb_rating)
FROM movies
WHERE year = 2000;
```

```sql
SELECT AVG(imdb_rating)
FROM movies
WHERE year = 2001;
```

We would have to write many statements.

## 4. Using GROUP BY

```sql
SELECT year,
AVG(imdb_rating)
FROM movies
GROUP BY year;
```

# XXXI. Group By II

## 1. GROUP BY Using Column Numbers

In SQL, we can use column numbers instead of column names.

Example:

```sql
SELECT category,
       price,
       AVG(downloads)
FROM fake_apps
GROUP BY category, price;
```

Can be written shorter as:

```sql
SELECT category,
       price,
       AVG(downloads)
FROM fake_apps
GROUP BY 1, 2;
```

## 2. Meaning of the Numbers

```sql
SELECT category,
       price,
       AVG(downloads)
```

Column order:

| Number | Column         |
| ------ | -------------- |
| 1      | category       |
| 2      | price          |
| 3      | AVG(downloads) |

So:

```sql
GROUP BY 1, 2
```

is equivalent to:

```sql
GROUP BY category, price
```

## 3. Why Use This?

It makes the statement shorter.

Example:

Instead of:

```sql
GROUP BY ROUND(imdb_rating)
ORDER BY ROUND(imdb_rating)
```

We can write:

```sql
GROUP BY 1
ORDER BY 1
```

## 4. Example from the Lesson

Original statement:

```sql
SELECT category,
       price,
       AVG(downloads)
FROM fake_apps
GROUP BY category, price;
```

Using numbers:

```sql
SELECT category,
       price,
       AVG(downloads)
FROM fake_apps
GROUP BY 1, 2;
```

## 5. Another Example

```sql
SELECT ROUND(imdb_rating),
       COUNT(name)
FROM movies
GROUP BY 1
ORDER BY 1;
```

Here:

```text
1 = ROUND(imdb_rating)
```

So SQL understands it as:

```sql
GROUP BY ROUND(imdb_rating)
ORDER BY ROUND(imdb_rating)
```

## Things to Remember

* You can use column numbers in `GROUP BY`.
* `1` = The first column in `SELECT`.
* `2` = The second column in `SELECT`.
* `3` = The third column in `SELECT`.

Example:

```sql
SELECT category,
       price,
       AVG(downloads)
FROM fake_apps
GROUP BY 1, 2;
```

Equivalent to:

```sql
GROUP BY category, price
```

## Lesson Summary

* `GROUP BY` can use column numbers instead of column names.
* This makes the statement shorter.
* `GROUP BY 1` means group by the first column in `SELECT`.
* `GROUP BY 1, 2` means group by the first and second columns in `SELECT`.
* This style is often used when the statement has long expressions like `ROUND(imdb_rating)`.

# XXXII. Having

## 1. What is HAVING?

`HAVING` is used to filter groups of data after `GROUP BY` has been applied.

Simply put:

* `WHERE` filters individual rows.
* `HAVING` filters groups.

## 2. Why Do We Need HAVING?

Suppose we have:

```sql
SELECT price,
       COUNT(*)
FROM fake_apps
GROUP BY price;
```

Result:

| price | COUNT(*) |
| ----- | -------- |
| 0     | 25       |
| 0.99  | 15       |
| 1.99  | 8        |
| 2.99  | 3        |

We only want to see price levels that have:

**More than 10 apps**

This is where `HAVING` is used.

## 3. Syntax

```sql
SELECT column_name,
       aggregate_function()
FROM table_name
GROUP BY column_name
HAVING condition;
```

## 4. How is HAVING Different from WHERE?

### WHERE

Filters individual rows of data.

Example:

```sql
SELECT *
FROM fake_apps
WHERE downloads > 20000;
```

SQL filters each app individually.

### HAVING

Filters groups of data.

Example:

```sql
SELECT price,
       COUNT(*)
FROM fake_apps
GROUP BY price
HAVING COUNT(*) > 10;
```

SQL filters the price groups.

## 5. Example from the Lesson

```sql
SELECT year,
       genre,
       COUNT(name)
FROM movies
GROUP BY 1, 2
HAVING COUNT(name) > 10;
```

Meaning:

### Step 1

```sql
GROUP BY 1, 2
```

Group by:

* year
* genre

### Step 2

```sql
COUNT(name)
```

Count the number of movies in each group.

### Step 3

```sql
HAVING COUNT(name) > 10
```

Keep only groups that contain more than 10 movies.

## 7. Order of Clauses

```text

FROM
WHERE
GROUP BY
HAVING
ORDER BY
LIMIT
```

Remember:

* `WHERE` comes before `GROUP BY`
* `HAVING` comes after `GROUP BY`
* `ORDER BY` comes after `HAVING`

## 8. When Should We Use HAVING?

When the condition involves:

* `COUNT()`
* `SUM()`
* `AVG()`
* `MAX()`
* `MIN()`

Examples:

```sql
HAVING COUNT(*) > 10
```

```sql
HAVING AVG(price) > 2
```

```sql
HAVING SUM(downloads) > 50000
```

## Things to Remember

* `WHERE` filters rows of data.
* `HAVING` filters groups of data.
* `HAVING` always comes after `GROUP BY`.
* `HAVING` is commonly used with:

  * `COUNT()`
  * `SUM()`
  * `AVG()`
  * `MAX()`
  * `MIN()`

Example:

```sql
SELECT price,
       COUNT(*)
FROM fake_apps
GROUP BY price
HAVING COUNT(*) > 10;
```

## Lesson Summary

* `HAVING` is used to filter groups of data after `GROUP BY`.
* Unlike `WHERE`, `HAVING` does not filter individual rows; it filters entire groups.
* `HAVING` is commonly used with aggregate functions such as `COUNT()`, `SUM()`, `AVG()`, `MAX()`, and `MIN()`.
* Example: `HAVING COUNT(*) > 10` keeps only groups that contain more than 10 records.
