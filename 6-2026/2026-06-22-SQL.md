# VII. Alter

## 1. What is ALTER TABLE?

`ALTER TABLE` is used to change the structure of an existing table.

Examples:

* Add a new column.
* Modify a column.
* Delete a column (supported by some databases).

In this lesson, we use it to add a new column.

## 2. Syntax

```sql
ALTER TABLE table_name
ADD COLUMN column_name data_type;
```

Meaning:

* `ALTER TABLE` = Modify a table.
* `table_name` = Table name.
* `ADD COLUMN` = Add a new column.
* `column_name` = Column name.
* `data_type` = Data type.

## 3. Example

```sql
ALTER TABLE celebs
ADD COLUMN twitter_handle TEXT;
```

Meaning:

* Modify the `celebs` table.
* Add a new column named `twitter_handle`.
* The data type is `TEXT`.

## 4. Statement Breakdown

### ALTER TABLE

```sql
ALTER TABLE
```

Used to change the structure of a table.

### celebs

```text
celebs
```

The name of the table to be modified.

### ADD COLUMN

```sql
ADD COLUMN
```

Used to add a new column.

### twitter_handle TEXT

```sql
twitter_handle TEXT
```

* `twitter_handle` = Column name.
* `TEXT` = Text data type.

## 5. Before ALTER TABLE

Table: `celebs`

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |

The table has 3 columns.

## 6. After ALTER TABLE

Run:

```sql
ALTER TABLE celebs
ADD COLUMN twitter_handle TEXT;
```

The table becomes:

| id | name          | age | twitter_handle |
| -- | ------------- | --- | -------------- |
| 1  | Justin Bieber | 29  | NULL           |
| 2  | Taylor Swift  | 34  | NULL           |

A new column has been added.

## 7. What is NULL?

`NULL` is a special value in SQL.

It means:

* No data exists yet.
* The data is unknown.

Example:

| id | name          | age | twitter_handle |
| -- | ------------- | --- | -------------- |
| 1  | Justin Bieber | 29  | NULL           |

This means:

Justin Bieber does not have any data in the `twitter_handle` column yet.

## 8. Viewing the Result

After adding the column:

```sql
ALTER TABLE celebs
ADD COLUMN twitter_handle TEXT;

SELECT * FROM celebs;
```

`SELECT *` will display the entire table, including the newly created column.

### Things to Remember

* `ALTER TABLE` is used to modify a table structure.
* `ADD COLUMN` is used to add a new column.
* `twitter_handle` is the new column name.
* `TEXT` is a text data type.
* `NULL` means there is no data.
* After adding a new column, existing rows will have `NULL` values in that column.

# VIII. Update

## 1. What is UPDATE?

`UPDATE` is used to modify existing data in a table.

When data is entered incorrectly or needs to be changed, we use `UPDATE`.

## 2. Syntax

```sql
UPDATE table_name
SET column_name = value
WHERE condition;
```

Meaning:

* `UPDATE` = Update data.
* `table_name` = Table name.
* `SET` = The column to modify and its new value.
* `WHERE` = Specifies which row(s) to update.

## 3. Example

```sql
UPDATE celebs
SET twitter_handle = '@taylorswift13'
WHERE id = 4;
```

Meaning:

* Modify the `celebs` table.
* Set a new value for the `twitter_handle` column.
* Only update the row where `id = 4`.

## 4. Statement Breakdown

### UPDATE

```sql
UPDATE celebs
```

Used to update data in the `celebs` table.

### SET

```sql
SET twitter_handle = '@taylorswift13'
```

Meaning:

* `twitter_handle` is the column being modified.
* `@taylorswift13` is the new value.

### WHERE

```sql
WHERE id = 4
```

Meaning:

Only update the row where:

```text
id = 4
```

## 5. Before UPDATE

| id | name         | age | twitter_handle |
| -- | ------------ | --- | -------------- |
| 4  | Taylor Swift | 34  | NULL           |

## 6. After UPDATE

Run:

```sql
UPDATE celebs
SET twitter_handle = '@taylorswift13'
WHERE id = 4;
```

Result:

| id | name         | age | twitter_handle |
| -- | ------------ | --- | -------------- |
| 4  | Taylor Swift | 34  | @taylorswift13 |

The data has been updated.

## 7. Why Use WHERE?

### With WHERE

```sql
UPDATE celebs
SET twitter_handle = '@taylorswift13'
WHERE id = 4;
```

Only the row with `id = 4` is modified.

### Without WHERE

```sql
UPDATE celebs
SET twitter_handle = '@taylorswift13';
```

All rows will be changed:

| id | name          | twitter_handle |
| -- | ------------- | -------------- |
| 1  | Justin Bieber | @taylorswift13 |
| 2  | Emma Watson   | @taylorswift13 |
| 3  | Tom Holland   | @taylorswift13 |

=> This is usually a very dangerous mistake.

## 8. Viewing the Result

After UPDATE:

```sql
UPDATE celebs
SET twitter_handle = '@taylorswift13'
WHERE id = 4;

SELECT * FROM celebs;
```

`SELECT *` will display the data after it has been modified.

### Things to Remember

* `UPDATE` is used to modify existing data.
* `SET` specifies the column and new value.
* `WHERE` identifies which row(s) should be updated.
* Without `WHERE`, all rows may be modified.
* `UPDATE` does not add new data; it only changes existing data.

# IX. Delete

## 1. What is DELETE?

`DELETE` is used to remove data from a table.

When a row of data is no longer needed, we use `DELETE`.

## 2. Syntax

```sql
DELETE FROM table_name
WHERE condition;
```

Meaning:

* `DELETE FROM` = Remove data.
* `table_name` = Table name.
* `WHERE` = Condition for selecting rows to delete.

## 3. Example

```sql
DELETE FROM celebs
WHERE twitter_handle IS NULL;
```

Meaning:

* Delete data from the `celebs` table.
* Only remove rows where `twitter_handle` is `NULL`.

## 4. Statement Breakdown

### DELETE FROM

```sql
DELETE FROM celebs
```

Used to remove data from the `celebs` table.

### WHERE

```sql
WHERE twitter_handle IS NULL
```

Selects rows that contain `NULL` values.

### IS NULL

```sql
IS NULL
```

Meaning:

The value is empty or no data exists.

## 5. Before DELETE

| id | name          | twitter_handle |
| -- | ------------- | -------------- |
| 1  | Justin Bieber | NULL           |
| 2  | Taylor Swift  | @taylorswift13 |
| 3  | Tom Holland   | NULL           |

## 6. After DELETE

Run:

```sql
DELETE FROM celebs
WHERE twitter_handle IS NULL;
```

Result:

| id | name         | twitter_handle |
| -- | ------------ | -------------- |
| 2  | Taylor Swift | @taylorswift13 |

Rows containing `NULL` values have been deleted.

## 7. Why Use WHERE?

### With WHERE

```sql
DELETE FROM celebs
WHERE twitter_handle IS NULL;
```

Only rows that satisfy the condition are deleted.

### Without WHERE

```sql
DELETE FROM celebs;
```

All data in the table will be deleted.

| id | name | twitter_handle |
| -- | ---- | -------------- |
|    |      |                |

=> The table still exists, but all data inside it is removed.

## 8. Viewing the Result

After DELETE:

```sql
DELETE FROM celebs
WHERE twitter_handle IS NULL;

SELECT * FROM celebs;
```

`SELECT *` will display the remaining data in the table.

### Things to Remember

* `DELETE` is used to remove data.
* `DELETE FROM` specifies the table from which data will be removed.
* `WHERE` is used to select the rows to delete.
* `IS NULL` is used to check for `NULL` values.
* Without `WHERE`, all data in the table will be deleted.
* `DELETE` removes data only; it does not delete the table.

# X. Constraints

## 1. What are Constraints?

Constraints are rules applied to columns in a table.

Purpose:

* Control the data being entered.
* Prevent invalid data.
* Keep data more accurate.

Constraints are written after the data type.

## 2. Example

```sql
CREATE TABLE celebs (
   id INTEGER PRIMARY KEY,
   name TEXT UNIQUE,
   date_of_birth TEXT NOT NULL,
   date_of_death TEXT DEFAULT 'Not Applicable'
);
```

This table has 4 columns, and each column has a different constraint.

## 3. PRIMARY KEY

```sql
id INTEGER PRIMARY KEY
```

Meaning:

* Every row must have a different id.
* Duplicate ids are not allowed.

Correct example:

| id | name          |
| -- | ------------- |
| 1  | Justin Bieber |
| 2  | Taylor Swift  |

Incorrect example:

| id | name          |
| -- | ------------- |
| 1  | Justin Bieber |
| 1  | Taylor Swift  |

=> Error because the id is duplicated.

## 4. UNIQUE

```sql
name TEXT UNIQUE
```

Meaning:

* Values in the column must be different.
* Duplicate values are not allowed.

Correct example:

| name          |
| ------------- |
| Justin Bieber |
| Taylor Swift  |

Incorrect example:

| name          |
| ------------- |
| Justin Bieber |
| Justin Bieber |

=> Error because the name is duplicated.

## 5. NOT NULL

```sql
date_of_birth TEXT NOT NULL
```

Meaning:

* Data is required.
* The column cannot be empty.

Correct example:

| date_of_birth |
| ------------- |
| 1994-03-01    |

Incorrect example:

| date_of_birth |
| ------------- |
| NULL          |

=> Error because this column cannot be empty.

## 6. DEFAULT

```sql
date_of_death TEXT DEFAULT 'Not Applicable'
```

Meaning:

* If no value is entered, SQL automatically inserts a default value.

Default value:

```text
Not Applicable
```

Example:

```sql
INSERT INTO celebs
(id, name, date_of_birth)
VALUES
(1, 'Justin Bieber', '1994-03-01');
```

`date_of_death` is not provided.

Result:

| id | name          | date_of_birth | date_of_death  |
| -- | ------------- | ------------- | -------------- |
| 1  | Justin Bieber | 1994-03-01    | Not Applicable |

SQL automatically inserts the default value.

## 7. Example from the Lesson

```sql
CREATE TABLE awards (
   id INTEGER PRIMARY KEY,
   recipient TEXT NOT NULL,
   award_name TEXT DEFAULT 'Grammy'
);
```

Meaning:

### id INTEGER PRIMARY KEY

* id is an integer.
* It cannot be duplicated.

### recipient TEXT NOT NULL

* recipient is text.
* Data is required.

### award_name TEXT DEFAULT 'Grammy'

* award_name is text.
* If no value is entered, it automatically receives the value Grammy.

## 8. Example of Inserting Data

```sql
INSERT INTO awards
(id, recipient)
VALUES
(1, 'Taylor Swift');
```

Result:

| id | recipient    | award_name |
| -- | ------------ | ---------- |
| 1  | Taylor Swift | Grammy     |

Because `award_name` was not provided, SQL automatically uses the default value.

### Things to Remember

* A constraint is a rule applied to a column.
* PRIMARY KEY = Cannot be duplicated.
* UNIQUE = Values must be different.
* NOT NULL = Cannot be empty.
* DEFAULT = Default value if no data is entered.
* Constraints help keep data accurate and prevent errors.

A query is a request for data from a database.

Simply put:

Query = Asking the database for the information you need.

Examples:

* What are all the movies in the database?
* Which movies were released in 2020?

Databases usually contain a large amount of data.

Example:

# XIII. As

## 1. What is AS?

`AS` is used to temporarily rename a column or table.

The new name is called an:

**Alias**

An alias only appears in the returned result.

It does not change the actual column name in the database.

## 2. Syntax

```sql
SELECT column_name AS 'alias_name'
FROM table_name;
```

Meaning:

* SELECT = Retrieve data.
* column_name = Column to retrieve.
* AS = Temporary rename.
* alias_name = New name.
* FROM = Which table to retrieve data from.

## 3. Example

```sql
SELECT name AS 'Titles'
FROM movies;
```

Meaning:

* Retrieve the `name` column.
* Rename the column to `Titles`.

## 4. Before Using AS

```sql
SELECT name
FROM movies;
```

Result:

| name    |
| ------- |
| Avatar  |
| Titanic |
| Frozen  |

The column name is:

```text
name
```

## 5. After Using AS

```sql
SELECT name AS 'Titles'
FROM movies;
```

Result:

| Titles  |
| ------- |
| Avatar  |
| Titanic |
| Frozen  |

The column name has changed to:

```text
Titles
```

## 6. Example from the Exercise

```sql
SELECT imdb_rating AS 'IMDb'
FROM movies;
```

Meaning:

* Retrieve the `imdb_rating` column.
* Rename it to `IMDb`.

Result:

| IMDb |
| ---- |
| 7.8  |
| 8.5  |
| 9.0  |

## 7. Important Notes

### AS Does Not Change the Actual Name in the Database

Example:

```sql
SELECT name AS 'Titles'
FROM movies;
```

Only the displayed result shows:

```text
Titles
```

The real column name in the table is still:

```text
name
```

### Aliases Should Be Placed in Single Quotes

In SQLite:

```sql
SELECT name AS 'Titles'
FROM movies;
```

This is the recommended style in the course.

### Things to Remember

* AS is used to temporarily rename a column or table.
* The new name is called an Alias.
* Aliases only appear in the returned result.
* They do not change the data or table structure.
* Syntax:

```sql
SELECT column_name AS 'alias_name'
FROM table_name;
```

* Example:

```sql
SELECT imdb_rating AS 'IMDb'
FROM movies;
```

# XIV. Distinct

## 1. What is DISTINCT?

`DISTINCT` is used to retrieve values that are not duplicated.

It automatically removes repeated values.

## 2. Why Do We Need DISTINCT?

Suppose a table has this data:

| genre  |
| ------ |
| Action |
| Comedy |
| Action |
| Drama  |
| Comedy |

If we use:

```sql
SELECT genre
FROM movies;
```

Result:

| genre  |
| ------ |
| Action |
| Comedy |
| Action |
| Drama  |
| Comedy |

There are many repeated values.

## 3. Using DISTINCT

```sql
SELECT DISTINCT genre
FROM movies;
```

Result:

| genre  |
| ------ |
| Action |
| Comedy |
| Drama  |

The repeated values have been removed.

## 4. Syntax

```sql
SELECT DISTINCT column_name
FROM table_name;
```

Meaning:

* `SELECT` = Retrieve data.
* `DISTINCT` = Only retrieve unique values.
* `column_name` = The column to check.
* `FROM` = From which table.

## 5. Example with genre

```sql
SELECT DISTINCT genre
FROM movies;
```

Meaning:

* Retrieve movie genres.
* Do not display duplicated genres.

Example result:

| genre     |
| --------- |
| Action    |
| Comedy    |
| Drama     |
| Horror    |
| Animation |

## 6. Example with year

```sql
SELECT DISTINCT year
FROM movies;
```

Meaning:

* Retrieve release years.
* Each year appears only once.

Example:

Original data:

| year |
| ---- |
| 2000 |
| 2001 |
| 2000 |
| 2005 |
| 2001 |

Result:

| year |
| ---- |
| 2000 |
| 2001 |
| 2005 |

## 7. When Should We Use DISTINCT?

`DISTINCT` is very useful when you want to know:

* What movie genres are there?
* What release years are there?
* What countries are there?
* What jobs are there?

And you do not want to see repeated values.

### Things to Remember

* `DISTINCT` is used to remove duplicated data.
* It only displays unique values.
* Syntax:

```sql
SELECT DISTINCT column_name
FROM table_name;
```

* Example:

```sql
SELECT DISTINCT genre
FROM movies;
```

* It can be used with:

  * genre
  * year
  * country
  * name
  * and many other columns

# XV. Where

## 1. What is WHERE?

`WHERE` is used to filter data based on a condition.

Simply put:

Only retrieve rows that satisfy the condition.

## 2. Why Do We Need WHERE?

Suppose the `movies` table has:

| name    | year | imdb_rating |
| ------- | ---- | ----------- |
| Movie A | 2010 | 9.0         |
| Movie B | 2018 | 7.5         |
| Movie C | 2020 | 8.5         |

If we use:

```sql
SELECT * FROM movies;
```

Result:

=> Display all movies.

But if we only want to see movies with an IMDb rating greater than 8, we need to use `WHERE`.

## 3. Syntax

```sql
SELECT *
FROM table_name
WHERE condition;
```

Meaning:

* `SELECT` = Retrieve data.
* `FROM` = From which table.
* `WHERE` = Filter condition.
* `condition` = Condition.

## 4. Example

```sql
SELECT *
FROM movies
WHERE imdb_rating > 8;
```

Meaning:

* Retrieve all data.
* Only display movies with an IMDb rating greater than 8.

Example result:

| name    | imdb_rating |
| ------- | ----------- |
| Movie A | 9.0         |
| Movie C | 8.5         |

Movie B does not appear because 7.5 is not greater than 8.

## 5. Operator =

```sql
WHERE year = 2020
```

Meaning:

Only retrieve movies released in 2020.

## 6. Operator !=

```sql
WHERE year != 2020
```

Meaning:

Retrieve all movies except movies from 2020.

## 7. Operator >

```sql
WHERE imdb_rating > 8
```

Meaning:

IMDb rating is greater than 8.

## 8. Operator <

```sql
WHERE imdb_rating < 5
```

Meaning:

IMDb rating is less than 5.

This is the example in the exercise.

## 9. Operator >=

```sql
WHERE imdb_rating >= 8
```

Meaning:

IMDb rating is greater than or equal to 8.

## 10. Operator <=

```sql
WHERE imdb_rating <= 5
```

Meaning:

IMDb rating is less than or equal to 5.

## 11. Examples from the Exercise

### Retrieve movies with IMDb rating below 5

```sql
SELECT *
FROM movies
WHERE imdb_rating < 5;
```

### Retrieve movies released after 2014

```sql
SELECT *
FROM movies
WHERE year > 2014;
```

Meaning:

* `year > 2014`
* Only retrieve movies released after 2014.

## 12. Quick Way to Read

### WHERE year > 2014

Read as:

Filter rows where `year` is greater than 2014.

### WHERE imdb_rating < 5

Read as:

Filter rows where `imdb_rating` is less than 5.

### Things to Remember

* `WHERE` is used to filter data.
* Only rows that satisfy the condition are displayed.
* Common operators:

| Operator | Meaning                  |
| -------- | ------------------------ |
| =        | Equal to                 |
| !=       | Not equal to             |
| >        | Greater than             |
| <        | Less than                |
| >=       | Greater than or equal to |
| <=       | Less than or equal to    |

* Example:

```sql
SELECT *
FROM movies
WHERE year > 2014;
```

# XVI. Like I

## 1. What is LIKE?

`LIKE` is used to search for data that matches a pattern.

Simply put:

`LIKE` helps find data based on a character pattern.

Examples:

* Find names that start with "A".
* Find movies that contain the word "Star".
* Find email addresses that end with ".com".

## 2. The Special Character _

The underscore character:

```text
_
```

means:

Represents exactly **one** arbitrary character.

Example:

```text
S_even
```

Can match:

```text
Seven
Se7en
```

Because there is exactly one character in the middle.

## 3. Syntax

```sql
SELECT *
FROM table_name
WHERE column_name LIKE 'pattern';
```

## 4. Example from the Lesson

Suppose the table contains:

| name    |
| ------- |
| Seven   |
| Se7en   |
| Titanic |

We want to find names that:

* Start with "Se"
* End with "en"
* Have exactly one character in the middle

We use:

```sql
SELECT *
FROM movies
WHERE name LIKE 'Se_en';
```

## 5. Breakdown

```text
'Se_en'
```

Meaning:

| Part | Meaning                 |
| ---- | ----------------------- |
| Se   | Starts with Se          |
| _    | One arbitrary character |
| en   | Ends with en            |

Matching values:

```text
Seven
Se7en
```

Not matching:

```text
Seen
Seeen
Season
```

Because they do not match the pattern `Se_en`.

## 6. Another Example

Find names with 4 characters that start with A:

```sql
WHERE name LIKE 'A___'
```

Matching examples:

```text
Anna
Alex
```

Each underscore represents one character.

### Things to Remember

* `LIKE` is used to search for data by pattern.
* `_` represents exactly one arbitrary character.
* Example:

```sql
WHERE name LIKE 'Se_en'
```

Will find:

```text
Seven
Se7en
```

* The number of underscores = the number of characters being replaced.

# XVII. Is Null

## 1. What is NULL?

`NULL` is a special value in SQL.

It means:

* No data exists.
* The data is unknown.

Example:

| name    | imdb_rating |
| ------- | ----------- |
| Avatar  | 8.5         |
| Titanic | 9.0         |
| Frozen  | NULL        |

The movie Frozen does not have an IMDb rating yet, so its value is `NULL`.

## 2. Why Can't We Use = NULL?

Incorrect:

```sql
WHERE imdb_rating = NULL
```

This is incorrect because SQL does not allow checking `NULL` using:

* `=`
* `!=`

## 3. IS NULL

`IS NULL` is used to find `NULL` values.

Syntax:

```sql
SELECT column_name
FROM table_name
WHERE column_name IS NULL;
```

## 4. Example

```sql
SELECT name
FROM movies
WHERE imdb_rating IS NULL;
```

Meaning:

* Retrieve the `name` column.
* Only retrieve movies without an IMDb rating.

Example data:

| name    | imdb_rating |
| ------- | ----------- |
| Avatar  | 8.5         |
| Frozen  | NULL        |
| Titanic | 9.0         |

Result:

| name   |
| ------ |
| Frozen |

## 5. IS NOT NULL

`IS NOT NULL` is used to find values that contain data.

Syntax:

```sql
SELECT column_name
FROM table_name
WHERE column_name IS NOT NULL;
```

## 6. Example

```sql
SELECT name
FROM movies
```

# XVIII. Between

## 1. What is BETWEEN?

`BETWEEN` is used to filter data within a range of values.

Simply put:

Retrieve data between value A and value B.

## 2. Syntax

```sql
SELECT *
FROM table_name
WHERE column_name BETWEEN value1 AND value2;
```

Meaning:

* `BETWEEN` = Within a range.
* `value1` = Starting value.
* `value2` = Ending value.

## 3. BETWEEN with Numbers

Example:

```sql
SELECT *
FROM movies
WHERE year BETWEEN 1990 AND 1999;
```

Meaning:

* Retrieve movies from 1990 to 1999.
* Includes both 1990 and 1999.

Example:

| name    | year |
| ------- | ---- |
| Movie A | 1990 |
| Movie B | 1995 |
| Movie C | 1999 |
| Movie D | 2000 |

Result:

| name    | year |
| ------- | ---- |
| Movie A | 1990 |
| Movie B | 1995 |
| Movie C | 1999 |

Movie D is not selected.

## 4. What is BETWEEN Equivalent To?

```sql
WHERE year BETWEEN 1990 AND 1999
```

is equivalent to:

```sql
WHERE year >= 1990
AND year <= 1999
```

## 5. BETWEEN with Text

`BETWEEN` can also be used with text.

Example:

```sql
SELECT *
FROM movies
WHERE name BETWEEN 'A' AND 'J';
```

Meaning:

* Retrieve movies whose names begin from A to J.

Example:

| name     |
| -------- |
| Avatar   |
| Batman   |
| Frozen   |
| Iron Man |
| J        |
| Jaws     |
| Titanic  |

Result:

| name     |
| -------- |
| Avatar   |
| Batman   |
| Frozen   |
| Iron Man |
| J        |

## 6. Important Note About Text

```sql
WHERE name BETWEEN 'A' AND 'J'
```

Includes:

```text
J
```

But does not include:

```text
Jaws
```

Because SQL only considers the upper limit to be the letter `"J"`.

## 9. When Should We Use BETWEEN?

Use it when filtering:

* Year ranges.
* Score ranges.
* Price ranges.
* Date ranges.
* Letter ranges.

### Things to Remember

* `BETWEEN` is used to filter data within a range.
* Syntax:


# XIX. And

## 1. What is AND?

`AND` is used to combine multiple conditions in a `WHERE` clause.

Simply put:

All conditions must be true.

## 2. Why Do We Need AND?

Suppose you want to find:

* Movies in the Horror genre.
* And released before 1985.

Using only one condition is not enough.

We need to use `AND`.

## 3. Syntax

```sql
SELECT *
FROM table_name
WHERE condition_1
AND condition_2;
```

Meaning:

* `condition_1` must be true.
* `condition_2` must also be true.

=> The row will be displayed only if both conditions are true.

## 4. Example

```sql
SELECT *
FROM movies
WHERE year BETWEEN 1990 AND 1999
AND genre = 'romance';
```

Meaning:

Condition 1:

```sql
year BETWEEN 1990 AND 1999
```

Movies released between 1990 and 1999.

Condition 2:

```sql
genre = 'romance'
```

Movies in the romance genre.

Result:

Only movies that:

* Are in the romance genre.
* And were released during the 1990s.

## 5. Understanding AND

Example:

| year | genre   |
| ---- | ------- |
| 1995 | romance |

* Both conditions are true.

=> Selected.

| year | genre  |
| ---- | ------ |
| 1995 | action |

* Not romance.

=> Not selected.

| year | genre   |
| ---- | ------- |
| 2005 | romance |

* Not within 1990–1999.

=> Not selected.
