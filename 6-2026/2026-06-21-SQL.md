# I. Introduction to SQL

## 1. What is SQL?

SQL (Structured Query Language) is a language used to work with data in a database.

SQL helps you:

* View data
* Add data
* Update data
* Delete data

## 2. What is a Database?

A database is a place where data is stored.

Examples:

* Facebook stores user information.
* Shopee stores products and orders.
* Schools store student information.

## 3. What is a Relational Database?

A relational database is a type of database that stores data in tables.

Example: **Students**

| id | name |
| -- | ---- |
| 1  | An   |
| 2  | Lan  |

In a table:

* Each row represents one record.
* Each column represents one type of information.

## 4. SQLite

This course uses **SQLite**.

SQLite is a Relational Database Management System (RDBMS).

Advantages:

* Lightweight
* Easy to learn
* No complex installation required

## 5. Your First SQL Command

```sql
SELECT * FROM celebs;
```

Meaning:

* `SELECT` = Retrieve data
* `*` = All columns
* `FROM celebs` = From the table named `celebs`

=> Display all data from the `celebs` table.

## 6. Example

Suppose the `celebs` table contains:

| id | name         |
| -- | ------------ |
| 1  | Taylor Swift |
| 2  | Emma Watson  |
| 3  | Tom Holland  |

Run:

```sql
SELECT * FROM celebs;
```

Result:

| id | name         |
| -- | ------------ |
| 1  | Taylor Swift |
| 2  | Emma Watson  |
| 3  | Tom Holland  |

=> All records in the table will be displayed.

### Things to Remember

* SQL is used to work with data.
* A database stores data.
* A relational database stores data in tables.
* SQLite is the database management system used in this course.
* `SELECT` retrieves data.
* `*` means all columns.
* `FROM` specifies the table to retrieve data from.

# II. Relational Databases

## 1. What is a Relational Database?

A relational database is a type of database that stores data in one or more tables.

Example:

A school may have the following tables:

* Students
* Courses
* Teachers

Each table stores a different type of data.

## 2. What is a Table?

A table is where data is stored.

Example: `celebs`

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |
| 3  | Tom Holland   | 28  |

This table stores information about celebrities.

## 3. What is a Column?

A column represents a type of information in a table.

Example:

| id | name | age |

Columns:

* id
* name
* age

Meaning:

* id = identifier
* name = person's name
* age = person's age

## 4. What is a Row?

A row is a single record of data.

Example:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |

This row contains:

* id = 1
* name = Justin Bieber
* age = 29

## 5. What is a Data Type?

Every piece of data in a database has a data type.

A data type tells the database what kind of data is being stored.

## 6. Common Data Types

### INTEGER

Whole numbers.

Examples:

```text
1
20
100
-5
```

### TEXT

Text data.

Examples:

```text
Justin Bieber
Taylor Swift
Hello World
```

### DATE

Date values.

Format:

```text
YYYY-MM-DD
```

Examples:

```text
2026-06-21
2025-01-10
```

### REAL

Decimal numbers.

Examples:

```text
3.14
8.5
99.99
```

## 7. Complete Example

| id | name | birthday   | score |
| -- | ---- | ---------- | ----- |
| 1  | An   | 2005-05-10 | 8.5   |

Data types:

* id → INTEGER
* name → TEXT
* birthday → DATE
* score → REAL

### Things to Remember

* Relational databases store data in tables.
* Table = data storage structure.
* Column = type of information.
* Row = record.
* Data Type = type of data.

Common data types:

* INTEGER = whole number
* TEXT = text
* DATE = date
* REAL = decimal number

# III. Statements

## 1. What is a Statement?

A statement is a complete SQL command.

Example:

```sql
SELECT * FROM celebs;
```

This is a statement.

## 2. Statement Rules

Every SQL statement must end with:

```sql
;
```

Correct:

```sql
SELECT * FROM celebs;
```

Incorrect:

```sql
SELECT * FROM celebs
```

Missing the semicolon.

## 3. CREATE TABLE Statement

Example:

```sql
CREATE TABLE table_name (
   column_1 data_type,
   column_2 data_type,
   column_3 data_type
);
```

This statement creates a new table.

## 4. What is a Clause?

A clause is a part of an SQL statement that performs a specific function.

Example:

```sql
CREATE TABLE
```

Meaning:

* CREATE = create
* TABLE = table

=> Create a new table.

Clauses are usually written in uppercase.

Examples:

```sql
SELECT
FROM
WHERE
CREATE TABLE
```

## 5. What is a Table Name?

A table name identifies the table that the statement works with.

Example:

```sql
CREATE TABLE students
```

`students` is the table name.

## 6. What is a Parameter?

A parameter is information passed into a statement.

Example:

```sql
(
   id INTEGER,
   name TEXT,
   age INTEGER
)
```

These parameters define:

* Column names
* Column data types

## 7. Complete Example

```sql
CREATE TABLE students (
   id INTEGER,
   name TEXT,
   age INTEGER
);
```

Meaning:

* CREATE TABLE = create a table
* students = table name
* id INTEGER = integer column
* name TEXT = text column
* age INTEGER = integer column

=> Creates a table named `students` with three columns.

## 8. Analyzing a Statement

```sql
SELECT * FROM celebs;
```

### Clauses

```sql
SELECT
FROM
```

### Table Name

```text
celebs
```

Meaning:

* SELECT = retrieve data
* * = all columns
* FROM celebs = from the celebs table

=> Display all data in the table.

## 9. One Line or Multiple Lines

One line:

```sql
SELECT * FROM celebs;
```

Multiple lines:

```sql
SELECT *
FROM celebs;
```

Both are exactly the same.

### Things to Remember

* A statement is a complete SQL command.
* Every statement ends with `;`
* A clause performs a specific function.
* A table name identifies the table.
* Parameters provide additional information.
* `CREATE TABLE` creates tables.
* `SELECT * FROM celebs;` displays all data from the table.

# IV. Create

## 1. What is CREATE?

`CREATE` is used to create a new object in a database.

In this lesson, we use `CREATE TABLE` to create tables.

## 2. Syntax

```sql
CREATE TABLE table_name (
   column_1 data_type,
   column_2 data_type,
   column_3 data_type
);
```

Meaning:

* CREATE TABLE = create a new table
* table_name = table name
* column = column name
* data_type = data type

## 3. Creating the celebs Table

```sql
CREATE TABLE celebs (
   id INTEGER,
   name TEXT,
   age INTEGER
);
```

Meaning:

Creates a table named `celebs` with three columns:

* id
* name
* age

## 4. Column Breakdown

### id INTEGER

```sql
id INTEGER
```

* id = column name
* INTEGER = whole number

Examples:

```text
1
2
3
100
```

### name TEXT

```sql
name TEXT
```

* name = column name
* TEXT = text data

Examples:

```text
Justin Bieber
Taylor Swift
Tom Holland
```

### age INTEGER

```sql
age INTEGER
```

* age = column name
* INTEGER = whole number

Examples:

```text
20
25
30
```

## 5. Table After Creation

```sql
CREATE TABLE celebs (
   id INTEGER,
   name TEXT,
   age INTEGER
);
```

Creates:

| id | name | age |
| -- | ---- | --- |
|    |      |     |
|    |      |     |
|    |      |     |

The table exists, but it contains no data yet.

## 6. Checking Whether the Table Exists

If you run:

```sql
SELECT * FROM celebs;
```

before creating the table:

```text

```

because the table does not exist.

After running:

```sql
CREATE TABLE celebs (
   id INTEGER,

   age INTEGER
);
```

the table is created successfully.

### Things to Remember

* `CREATE TABLE` creates a new table.
* `celebs` is the table name.
* `id` is an INTEGER column.
* `name` is a TEXT column.
* `age` is an INTEGER column.
* A newly created table contains no data.

# V. Insert

## 1. What is INSERT?

`INSERT` is used to add new data into a table.

After creating a table with `CREATE TABLE`, the table is empty.

To add records, use `INSERT`.

## 2. Syntax

```sql
INSERT INTO table_name (column_1, column_2, column_3)
VALUES (value_1, value_2, value_3);
```

Meaning:

* INSERT INTO = add data to a table
* table_name = table name
* columns = columns receiving data
* VALUES = values to insert

## 3. Example

```sql
INSERT INTO celebs (id, name, age)
VALUES (1, 'Justin Bieber', 29);
```

Meaning:

Add a new row:

* id = 1
* name = Justin Bieber
* age = 29

## 4. Statement Breakdown

### INSERT INTO

```sql
INSERT INTO
```

Adds data into a table.

### celebs

```text
celebs
```

The table name.

### (id, name, age)

```sql
(id, name, age)
```

Columns receiving data.

### VALUES

```sql
VALUES
```

Introduces the data being inserted.

### (1, 'Justin Bieber', 29)

```sql
(1, 'Justin Bieber', 29)
```

Inserted values:

* 1 → id
* Justin Bieber → name
* 29 → age

## 5. Before INSERT

Table `celebs`:

| id | name | age |
| -- | ---- | --- |
|    |      |     |

The table is empty.

## 6. After INSERT

Run:

```sql
INSERT INTO celebs (id, name, age)
VALUES (1, 'Justin Bieber', 29);
```

Result:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |

One row has been added.

## 7. Adding More People

```sql
INSERT INTO celebs (id, name, age)
VALUES (2, 'Taylor Swift', 34);

INSERT INTO celebs (id, name, age)
VALUES (3, 'Tom Holland', 28);
```

Result:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |
| 3  | Tom Holland   | 28  |

## 8. Important Notes

### Integers Do Not Need Quotes

Correct:

```sql
29
```

Incorrect:

```sql
'29'
```

### Text Must Use Single Quotes

Correct:

```sql
'Justin Bieber'
```

Incorrect:

```sql
Justin Bieber
```

### Things to Remember

* `INSERT` adds data to a table.
* `INSERT INTO` specifies the table.
* `VALUES` contains the data being inserted.
* TEXT values must be enclosed in single quotes `' '`.
* INTEGER values do not need quotes.
* Each `INSERT` statement usually adds one row of data.

# VI. Select

## 1. What is SELECT?

`SELECT` is used to retrieve (view) data from a database.

Whenever you want to view data stored in a table, you use `SELECT`.

## 2. Basic Syntax

```sql
SELECT column_name
FROM table_name;
```

Meaning:

* `SELECT` = Retrieve data
* `column_name` = The column you want to view
* `FROM` = Specifies the table
* `table_name` = Name of the table

## 3. Example

```sql
SELECT name FROM celebs;
```

Meaning:

* Retrieve data from the `name` column
* From the `celebs` table

Suppose the table contains:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |
| 3  | Tom Holland   | 28  |

Result:

| name          |
| ------------- |
| Justin Bieber |
| Taylor Swift  |
| Tom Holland   |

Only the `name` column is displayed.

## 4. Statement Breakdown

```sql
SELECT name FROM celebs;
```

### SELECT

```sql
SELECT
```

Used to retrieve data.

### name

```sql
name
```

The column you want to view.

### FROM celebs

```sql
FROM celebs
```

Retrieves data from the `celebs` table.

## 5. Viewing All Columns

```sql
SELECT * FROM celebs;
```

Meaning:

* `SELECT` = Retrieve data
* `*` = All columns
* `FROM celebs` = From the `celebs` table

## 6. Example of SELECT *

Suppose the table contains:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |

Run:

```sql
SELECT * FROM celebs;
```

Result:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |

All data in the table is displayed.

## 7. What Does * Mean?

The `*` symbol is a wildcard character.

It means:

**All columns**

Instead of writing:

```sql
SELECT id, name, age FROM celebs;
```

you can write the shorter version:

```sql
SELECT * FROM celebs;
```

Both statements return the same columns.

## 8. What is a Result Set?

After a `SELECT` statement is executed, SQL creates a new table of results.

This table is called a **Result Set**.

A Result Set:

* Shows the requested data
* Is temporary
* Does not change the original data in the database

### Example

Original table:

| id | name          | age |
| -- | ------------- | --- |
| 1  | Justin Bieber | 29  |
| 2  | Taylor Swift  | 34  |

Query:

```sql
SELECT name FROM celebs;
```

Result Set:

| name          |
| ------------- |
| Justin Bieber |
| Taylor Swift  |

The original table remains unchanged.

---

## Things to Remember

* `SELECT` is used to retrieve data.
* `FROM` specifies the table to retrieve data from.
* You can retrieve a single column:

```sql
SELECT name FROM celebs;
```

* You can retrieve all columns:

```sql
SELECT * FROM celebs;
```

* `*` means all columns.
* The result returned by a `SELECT` statement is called a **Result Set**.
* A Result Set does not modify the original data in the database.
