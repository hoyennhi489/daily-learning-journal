## **SERVER AND WEB SERVER**

#### 1. What is a Server?

‚Ä¢ A **server** is a dedicated computer that runs applications and services such as:
o Websites
o Instant messaging
‚Ä¢ It is called a ‚Äúserver‚Äù because it provides services to other computers and users, called **clients**.

#### 2. Data Center

‚Ä¢ Servers are usually stored in **Data Centers**, which house hundreds or thousands of servers, all connected to the Internet.
‚Ä¢ A data center is designed to:
o Maintain a continuous power supply
o Ensure stable Internet connectivity
o Keep temperature consistent 24/7 to prevent overheating
‚Ä¢ There are many data centers around the world, allowing users to access content faster by connecting to the nearest one.

#### 3. Structure and Purpose of a Server

‚Ä¢ Servers are built according to their intended purpose:
o If used for image storage ‚Üí needs large hard drive capacity.
o If used for complex computations ‚Üí needs a powerful CPU and large RAM.
‚Ä¢ These components are called **Server Hardware**.

#### 4. Hardware and Software

| Comparison      | Hardware                                                                 | Software                                    |
| --------------- | ------------------------------------------------------------------------ | ------------------------------------------- |
| Definition      | The physical components of a server (CPU, RAM, hard drive, etc.)         | The code that runs on the hardware          |
| Characteristics | Tangible, difficult to modify because it requires real component changes | Easy to modify, can be updated or rewritten |
| Memory Tip      | ‚ÄúHard‚Äù = hard to change                                                  | ‚ÄúSoft‚Äù = easy to change                     |

#### 5. What is a Web Server?

‚Ä¢ A **web server** is a type of **service software** running on a server.
‚Ä¢ Main functions include:
o Storing and managing websites
o Storing data
o Securing the system
o Managing emails

#### 6. Main Functions of a Web Server

‚Ä¢ A web server processes requests from web browsers (web requests).
‚Ä¢ When a user opens a browser and enters a website address (e.g., [www.example.com](http://www.example.com)):
o The browser (client) sends a **request** to the web server.
o The web server processes it and sends a **response** containing the website‚Äôs content back to the browser for display.
‚Ä¢ This is called the **Request‚ÄìResponse Cycle**.

#### 7. Web Server Performance

‚Ä¢ Web servers are designed to:
o Handle thousands of requests per second from many different clients.
o Operate quickly, reliably, and securely.

#### 8. Summary

| Concept                | Quick Reminder                                                            |
| ---------------------- | ------------------------------------------------------------------------- |
| Server                 | A computer providing services to other computers (clients).               |
| Data Center            | Place housing many servers with 24/7 power, Internet, and cooling.        |
| Hardware               | Physical parts of a server ‚Äì hard to change.                              |
| Software               | Code running on a server ‚Äì easy to change.                                |
| Web Server             | Software managing websites, data, and security.                           |
| Request‚ÄìResponse Cycle | Process where the client sends a request and the server sends a response. |

#### 9. Study Notes

‚Ä¢ Hardware is tangible ‚Äî changing it means replacing physical parts.
‚Ä¢ Software is code ‚Äî it can be updated or edited easily.
‚Ä¢ A web server enables a website to function by responding to user requests.
‚Ä¢ When you enter a web address, the server sends data back for your browser to display.

## **WEBPAGE AND WEBSITE ‚Äì TECHNOLOGIES BEHIND WEB PAGES**

#### 1. Difference Between Webpage and Website

‚Ä¢ A **webpage** is a single document displaying text, images, videos, and other content in a browser.
‚Ä¢ A **website** is a collection of interlinked webpages.

‚Ä¢ The **homepage** contains links to many articles.
‚Ä¢ Clicking a link opens a new **webpage**.
‚Ä¢ All pages share similar addresses in the browser‚Äôs address bar, indicating they belong to the same website.

#### 2. The Scale of the Internet and Website Growth

‚Ä¢ Hundreds of thousands of new websites are created every day.
‚Ä¢ Fun comparison:

‚Ä¢ Despite billions of webpages, all are built on **three core technologies**.

‚Ä¢ A webpage is essentially a text document viewable with any text editor.
‚Ä¢ However, web developers use specialized tools for development.
‚Ä¢ To build webpages, you need three core technologies:

| JavaScript | ‚Äì                         | Creates interactivity, handles data, actions, and dynamic updates                   |

#### 5. Simple Analogy

‚Ä¢ **HTML** = the building ‚Äì structure, walls, floors, doors.
‚Ä¢ **CSS** = the decoration ‚Äì design, colors, layout.
‚Ä¢ **JavaScript** = the business operations ‚Äì customers, staff, and activities inside.

#### 6. Details of Each Technology

**a. HTML (HyperText Markup Language)**
‚Ä¢ HTML works using **markup tags**.
‚Ä¢ Tags describe content displayed in the browser:
o Headings

‚Ä¢ The way HTML marks up content is called **markup**.

**b. CSS (Cascading Style Sheets)**

‚Ä¢ When a webpage is sent from a web server to a browser:

‚Ä¢ This process is called **Page Rendering**.

## **WEB BROWSER AND THE REQUEST‚ÄìRESPONSE CYCLE**

#### 1. What is a Web Browser?

‚Ä¢ A **Web Browser** is an application used to navigate the World Wide Web.
‚Ä¢ It sends **requests** to web servers and receives **responses** containing webpage data, which it displays on your screen.
Examples: Chrome, Firefox, Safari, Edge.

#### 2. Entering a Web Address (URL)

‚Ä¢ The address bar in a browser is where you enter a website‚Äôs address.
‚Ä¢ This address is called a **URL (Uniform Resource Locator)**.

Example:
`https://www.example.com/about`
‚Äì `https` ‚Üí protocol (usually HTTP or HTTPS)
‚Äì `www.example.com` ‚Üí domain name
‚Äì `/about` ‚Üí file path

#### 3. HTTP Protocol and Communication Process

‚Ä¢ Browsers and servers exchange data via **HTTP (Hypertext Transfer Protocol)**.
‚Ä¢ This exchange follows the **Request‚ÄìResponse Cycle**:
Browser sends a request ‚Üí Server processes it ‚Üí Server responds ‚Üí Browser displays the result.

#### 4. Example: Searching the Internet

**Step 1:** Open your browser (e.g., Chrome or Firefox).
**Step 2:** Type the search engine‚Äôs address (e.g., Google or Bing).
**Step 3:** The browser sends a **request** to the search engine‚Äôs web server.
‚Ä¢ The web server processes the request and sends back a **webpage**.
**Step 4:** The browser receives the response and **renders** the search results.

#### 5. Interacting with a Webpage

When you type ‚Äúrestaurants near me‚Äù and press Search:

1. The browser sends a **new request** with the search query.
2. The server processes it and retrieves data from the **database**.
3. The server sends back a **new response** containing search results.
4. The browser renders a new webpage showing restaurant names, maps, reviews, etc.

#### 6. Request‚ÄìResponse Cycle Summary

1. User enters a URL or performs an action (e.g., clicking Search).
2. Browser creates and sends an **HTTP Request**.
3. Web Server processes it, querying the **Database** if needed.
4. Web Server sends an **HTTP Response**.
5. Browser **renders** the content.

This cycle happens every time you browse, stream, chat, or share files.

#### 7. Roles of Each Component

| Component              | Role                                                             |
| ---------------------- | ---------------------------------------------------------------- |
| Browser                | Sends requests and displays web content.                         |
| Web Server             | Receives requests, processes data, and sends responses.          |
| Database               | Stores data such as articles, user accounts, images, or reviews. |
| HTTP Protocol          | Communication rules between browser and server.                  |
| Request‚ÄìResponse Cycle | The data exchange loop between both sides.                       |

#### 8. Conclusion

‚Ä¢ A web browser is the bridge between users and the Internet.
‚Ä¢ All web browsing depends on the **HTTP Request‚ÄìResponse Cycle**.
‚Ä¢ The browser and server ‚Äútalk‚Äù via the Internet to transfer data.
‚Ä¢ Every search, video, or message involves thousands of data exchanges in seconds.

## **WEB HOSTING ‚Äì TYPES OF WEBSITE HOSTING**

#### 1. What is Web Hosting?

‚Ä¢ Websites and their files are stored on **web servers** in **data centers**.
‚Ä¢ Developers can put their websites online through **web hosting services** instead of owning physical hardware.
‚Ä¢ **Web hosting** is a service that lets you store your website‚Äôs files on a hosting provider‚Äôs server.
‚Ä¢ You rent storage space so your website can run reliably and securely.
‚Ä¢ Both individuals and businesses can use hosting services.

#### 2. Common Types of Web Hosting

**a. Shared Hosting**
‚Ä¢ Cheapest and most popular option.
‚Ä¢ Many websites share the same server and resources (CPU, memory, bandwidth).
‚Ä¢ If one site uses too many resources, others slow down.
‚Ä¢ Suitable for small websites with low traffic.
‚Ä¢ Often used by beginners or for testing.
‚Ä¢ Some free plans exist but usually include ads and limits.

**b. VPS Hosting (Virtual Private Server)**
‚Ä¢ A virtual server with dedicated resources like CPU, RAM, and bandwidth.
‚Ä¢ Several VPS instances run on one physical server, but each is isolated.
‚Ä¢ More stable and powerful than shared hosting, but also more expensive.
‚Ä¢ Suitable for medium websites or small businesses.

**c. Dedicated Hosting**
‚Ä¢ You own an entire physical server.
‚Ä¢ All resources belong exclusively to you.
‚Ä¢ High performance and reliability ‚Äì ideal for large websites or enterprise systems.
‚Ä¢ The most expensive option.

**d. Cloud Hosting**
‚Ä¢ Your website runs on a cloud network of multiple physical and virtual servers.
‚Ä¢ If one fails, another automatically takes over, ensuring uptime.
‚Ä¢ Resources can scale up or down based on demand.
‚Ä¢ You pay only for what you use.
‚Ä¢ Ideal for fast-growing or high-traffic websites.
‚Ä¢ Most modern web apps use this model.

#### 3. Pros and Cons

| Type              | Advantages                       | Disadvantages                     | Best For                       |
| ----------------- | -------------------------------- | --------------------------------- | ------------------------------ |
| Shared Hosting    | Cheap, easy to use               | Low performance, shared resources | Personal sites, small blogs    |
| VPS Hosting       | Stable, customizable             | More expensive                    | Small business sites           |
| Dedicated Hosting | Full control, strong performance | Highest cost                      | Large or critical systems      |
| Cloud Hosting     | Scalable, flexible, reliable     | Can be costly at high usage       | Large apps, fast-growing sites |

## **INTERNET PROTOCOL (IP) ‚Äì HOW DATA TRAVELS THROUGH THE INTERNET**

#### 1. Understanding IP

‚Ä¢ Like a home address, an **IP address** tells computers where to send data.
‚Ä¢ When you visit a website, your computer sends ‚Äúdata letters‚Äù to the server‚Äôs IP address.
‚Ä¢ IP ensures the data travels correctly across the Internet.

#### 2. What is an IP Address?

‚Ä¢ Every device connected to a network has a unique IP.
‚Ä¢ Two main types:
o **IPv4:** e.g., 192.0.2.235
o **IPv6:** e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334
‚Ä¢ IPv6 was created because IPv4 addresses are running out.

#### 3. Data Packets

‚Ä¢ Data is broken into small **packets** before transmission.
‚Ä¢ Each packet includes:
o **Header:** sender and receiver addresses.
o **Payload:** actual content (like the letter inside an envelope).

#### 4. Possible Transmission Errors

‚Ä¢ Packets may arrive late, out of order, or get lost.
‚Ä¢ Internet protocols handle these issues automatically.

#### 5. Two Main Transmission Methods

| Protocol                            | How It Works                                                          | Used For                                |
| ----------------------------------- | --------------------------------------------------------------------- | --------------------------------------- |
| TCP (Transmission Control Protocol) | Slower but reliable, ensures all data arrives correctly and in order. | Email, websites, file downloads         |
| UDP (User Datagram Protocol)        | Very fast, but doesn‚Äôt check for errors.                              | Video calls, live streams, online games |

#### 6. Quick Summary

‚Ä¢ IP = device ‚Äúhome address.‚Äù
‚Ä¢ Data is split into packets before sending.
‚Ä¢ TCP ensures accuracy; UDP prioritizes speed.
‚Ä¢ Thanks to IP and these protocols, the Internet runs smoothly and efficiently.

## **HTTP & HTTPS ‚Äì WEB COMMUNICATION PROTOCOLS**

#### 1. What is HTTP?

‚Ä¢ **HTTP (Hypertext Transfer Protocol)** is the communication protocol of the web, enabling browsers and servers to exchange data.
‚Ä¢ Every time you load a webpage, your browser sends a **request** and receives a **response**.
‚Ä¢ This **request‚Äìresponse cycle** is the foundation of all Internet communication.

#### 2. Structure of an HTTP Request

An HTTP Request includes four main parts:

1. **Method** ‚Äì describes the action:
   o GET ‚Äì retrieve data from the server.
   o POST ‚Äì send data to the server (e.g., a registration form).
   o PUT ‚Äì update existing data.
   o DELETE ‚Äì remove data.
2. **Path** ‚Äì resource path (e.g., `/images/photo.jpg`).
3. **Version** ‚Äì HTTP version (usually 1.1 or 2.0).
4. **Headers** ‚Äì additional info (browser type, language, content type, etc.).
   ‚Üí With POST/PUT, there‚Äôs also a **Body** containing sent data.

#### 3. Structure of an HTTP Response

Includes:
‚Ä¢ **Status line:** status code + message.
‚Ä¢ **Headers:** info about the server or returned data.
‚Ä¢ **Body:** actual content (HTML, image, JSON, etc.).

**Status Codes** are three-digit numbers (100‚Äì599) showing the request result.

#### 4. HTTP Status Code Groups

| Group | Range   | Meaning       | Example                                                         |
| ----- | ------- | ------------- | --------------------------------------------------------------- |
| 1xx   | 100‚Äì199 | Informational | 100 Continue                                                    |
| 2xx   | 200‚Äì299 | Success       | 200 OK                                                          |
| 3xx   | 300‚Äì399 | Redirection   | 301 Moved Permanently, 302 Found                                |
| 4xx   | 400‚Äì499 | Client Error  | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found |
| 5xx   | 500‚Äì599 | Server Error  | 500 Internal Server Error                                       |

#### 5. HTTPS ‚Äì The Secure Version of HTTP

‚Ä¢ **HTTPS (HTTP Secure)** = HTTP + encryption.
‚Ä¢ Protects transmitted information (e.g., passwords, credit cards).
‚Ä¢ Data is encrypted into ‚Äúsecret code,‚Äù readable only by the intended receiver.
‚Ä¢ Identified by a **lock icon üîí** in the browser‚Äôs address bar.

#### 6. Quick Summary

‚Ä¢ HTTP is the communication language between browsers and servers.
‚Ä¢ A Request includes method, path, version, headers, and possibly body.
‚Ä¢ A Response includes status code, headers, and body.
‚Ä¢ There are five main groups of status codes.
‚Ä¢ HTTPS ensures safe, encrypted data transmission.
